# ambient-bureaucracy

> 生成AI時代の **「新官僚主義（お役所仕事の逆説）」** と **「環境知能（アンビエントAI）」** をめぐる体系書のためのリポジトリ。
> ノウハウの発信は生成AIによって陳腐化した。ここで書くのは、**世界のAIリーダーたちの一次発言をエビデンスの骨格に据えた、まだ世に浸透していない世界観そのもの**である。

## この本のテーゼ（骨子）

1. DXは「脱お役所仕事」を掲げ、個人の自律を高めた。しかしその帰結は、一人ひとりの脳内認知負荷の爆発だった（ブルシットジョブの真の苦痛）。
2. 生成AIが最も得意とするのは、構造化されたプロセスである。
3. ゆえにAI時代の最適解は、**かつて捨てた「お役所仕事」（ガチガチのプロセス・バックログ・決済フロー）の構造にAIを刺し、AIに肩代わりさせて人間を苦役から解放する「新官僚主義の調和」**である。
4. ただしテクノロジーが現場に本当にリーチする唯一の条件は、**人間側の物理動線を1ミリも変えさせず、裏で勝手に完了させておく「環境知能（アンビエントAI）」**である（＝介護の手袋の逆説）。
5. この世界観を、Anthropic・NVIDIA・OpenAI 等のリーダー／研究者の**一次発言で建てる**。自分の言葉だけで組んでも説得力がないから。

## リポジトリ構成

```
.
├── README.md                     # この本の趣旨と歩き方
├── docs/
│   ├── thesis.md                 # 中核の主張（本の背骨・単一の真実）＋新官僚主義→環境知能の橋
│   ├── outline.md                # 章立て骨子（ストーリーライン）
│   ├── reader.md                 # 読者ペルソナ（翻訳の宛先＝DX信者の同僚T）
│   ├── backlog.md                # 制作バックログ（新官僚主義のドッグフーディング）
│   └── roadmap.md                # 拡充ロードマップ（Phase 0-5）
├── evidence/
│   ├── README.md                 # エビデンス採取プロトコル（一次発言の記録ルール）
│   └── ai-leaders-index.md       # 引用候補（AIリーダー／論文）の索引・埋めるべき空欄
└── books/ambient-bureaucracy/    # Zenn本（zenn-cli 互換）
    ├── config.yaml
    └── *.md                      # 各章
```

## 執筆ワークフロー（体験学習主義／小さく積む）

1. **主張は `docs/thesis.md` に1本だけ**置く。章はすべてこの背骨に接続する。
2. **章を書く前に、その章を支える一次発言を `evidence/` に採取**する（クレーム→引用→出典URL→一次/二次を必ずセットで）。
3. 章 (`books/.../*.md`) は、採取済みエビデンスを地の文で編み直す。**自分の意見はエビデンスの後に置く**。
4. `npx zenn preview` でプレビュー、`published: false` のまま骨子が固まるまで非公開で育てる。

## Zenn での執筆・公開

```bash
npm install            # zenn-cli を導入
npx zenn preview       # http://localhost:8000 でプレビュー
```

`books/ambient-bureaucracy/config.yaml` の `published: false` を `true` にすると公開対象になる（Zenn連携時）。
