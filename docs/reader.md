# 読者ペルソナ（翻訳の宛先）

> 本書の各章は、この**たった1人**に向けて書く。抽象的な「読者」に逃げない。
> 完了基準は「この人が"お役所仕事に戻す"に半歩うなずくか」。彼が脱落したら、その章は失敗。

## メインペルソナ：DXを信じてきた同僚「T」

- **属性**：40代、社内DX推進を数年リードしてきた中堅。オムロン的な製造業DXの現場。非エンジニア寄りだが技術リテラシーは高い。
- **信じている世界（＝先に肯定すべき前提）**：
  - お役所仕事＝悪。決裁フロー・様式・回覧は撲滅すべき非効率。
  - 「脱・属人化」「権限移譲」「会議削減」「アジャイル」こそ正義。自分はそれを推進してきた誇りがある。
- **口癖**：「それって前に戻すだけでは？」「せっかく簡素化したのに」「現場が増える施策は通らない」。
- **抵抗ポイント（なぜ"戻す"に反発するか）**：
  - 自分の10年の仕事（プロセス撲滅）を否定されたように感じる。
  - 「お役所仕事に戻す＝時代逆行」というフレームでしか受け取れない。
- **彼が動く条件（＝翻訳が刺さる瞬間）**：
  - 「あなたは間違っていた」ではなく「**あなたの努力の"意味"が、AI時代に変わった**」と示されたとき。
  - 消したプロセスが「AIの燃料（固めた文脈）」だったと、**自分の実感**（AIに雑に頼むと雑な答えが返る体験）に接続されたとき。

## 翻訳の鉄則（このペルソナ相手に）

1. **肯定を先に、短く**：彼の推進してきた世界を1〜2文で正確に肯定する（長くすると退屈で緊張が生まれない）。
2. **逆説を1文で撃つ**：「あなたが消したものこそ、AIが飢える固めた文脈だった」。ここが章の心臓。
3. **機構は"証拠"として従属**：生成AIの仕組み（入力の質＝出力の質）は、逆説の後に**根拠として**置く。先に出すと講義になって脱落する（道場#1の失敗）。
4. **"戻す"を再定義**：戻すのは**苦役**ではない。苦役はAIが肩代わりする。人間に戻るのは**構造**だけ。
